1. Présentation générale

Le backend est une API REST Spring Boot 3.2 sécurisée avec JSON Web Tokens (JWT). Il expose des endpoints pour deux types d'acteurs :

• Administrateur : CRUD complet sur Entreprises, Actionnaires, Résolutions, Assemblées Générales
• Actionnaire (USER) : consultation de ses AGs et expression de votes (POUR / CONTRE / NEUTRE)

La base de données utilisée est MySQL, gérée via Spring Data JPA / Hibernate avec création automatique des tables (spring.jpa.hibernate.ddl-auto=update).

🏗️
Architecture 3 couches
Controller → Service → Repository. Séparation claire des responsabilités.
🔐
Sécurité JWT
Authentification stateless. Token signé HMAC-SHA256, validité 24h. Rôles ADMIN / USER via Spring Security.
🗄️
6 entités JPA
User, Entreprise, Actionnaire, Resolution, AssembleeGenerale, Vote — avec relations ManyToOne et ManyToMany.
📦
Pattern DTO
8 DTOs pour ne jamais exposer les entités JPA directement. Contrôle fin des données envoyées au client.

2. Architecture en couches

🌐 Couche Client
Frontend React — envoie des requêtes HTTP avec header Authorization: Bearer {JWT}
↕
🎮 Controller
Reçoit les requêtes HTTP, valide le JWT, délègue au Service. Retourne les DTO en JSON.
↕
⚙️ Service
Logique métier. Convertit Entity ↔ DTO. Gère les transactions. Appelle les Repository.
↕
🗃️ Repository
Interfaces Spring Data JPA — génère automatiquement les requêtes SQL.
↕
🐬 MySQL
Base de données relationnelle. Tables créées/mises à jour automatiquement par Hibernate.

🏢 Entreprise

Champ Type Java Description

👥 Actionnaire

Table de liaison User ↔ Entreprise avec le nombre d'actions détenues.

Champ Type Java Relation Description
id Long PK Identifiant unique

Champ Type Java Description

🗳️ AssembleeGenerale

Entité centrale du système — relie une entreprise à un ensemble de résolutions.

Champ Type Java Relation Description

description String (TEXT) — Contexte et objectifs

Champ Type Java Description

5. Sécurité — JWT + Spring Security

🔑 JwtUtil.java

Méthode Entrée Sortie Description
generateToken() username, role String JWT Crée un token signé HMAC-SHA256, valide 24h
getUsernameFromToken() String token String username Extrait le subject (username) du payload JWT
validateToken() String token boolean Vérifie signature + expiration (retourne false si invalide) 
// Structure d'un token JWT généré :
// Header: {"alg":"HS256","typ":"JWT"}
// Payload: {"sub":"admin","role":"ADMIN","iat":...,"exp":...}
// Signature: HMAC-SHA256(base64(header)+"."+base64(payload), secretKey)

@Value("${app.jwt.secret}")  // clé secrète dans application.properties
private String jwtSecret;

@Value("${app.jwt.expiration}") // 86400000 ms = 24 heures
private long jwtExpiration;

🔒 JwtAuthFilter.java

Filtre HTTP exécuté avant chaque requête. Intercepte le header Authorization, valide le token et charge l'utilisateur dans le contexte Spring Security.

Requête HTTP
→
Lecture header Authorization
→
validateToken()
→
getUsernameFromToken()
→
loadUserByUsername()
→
SecurityContextHolder.set()

⚙️ SecurityConfig.java

Route Accès autorisé
/api/auth/** Public (login, register)
/api/admin/** ROLE_ADMIN uniquement
/api/user/** ROLE_USER uniquement
/uploads/** Public (accès aux logos)
Toutes autres routes Authentifié 
// Configuration CSRF désactivée (API REST stateless)
// Session STATELESS — chaque requête porte son propre token JWT
// Encodage BCrypt pour les mots de passe (force=10)
http
  .sessionManagement(s -> s.sessionCreationPolicy(STATELESS))
  .addFilterBefore(jwtAuthFilter, UsernamePasswordAuthenticationFilter.class)
  .authorizeHttpRequests(auth -> auth
    .requestMatchers("/api/auth/**").permitAll()
    .requestMatchers("/api/admin/**").hasRole("ADMIN")
    .requestMatchers("/api/user/**").hasRole("USER")
    .anyRequest().authenticated()
  );

⚙️ VoteService.java

Logique de vote avec contrainte d'unicité (un actionnaire ne vote qu'une fois par résolution) :

@Transactional
public VoteDTO vote(VoteRequest request, String username) {
    // 1. Charger l'utilisateur connecté
    User user = userRepo.findByUsername(username).orElseThrow(...);

    // 2. Trouver son lien actionnaire avec l'entreprise de la résolution
    Actionnaire actionnaire = actionnaireRepo
        .findByUserAndEntreprise(user, resolution.getEntreprise())
        .orElseThrow(() -> new RuntimeException("Pas actionnaire"));

    // 3. Chercher un vote existant (UPDATE si existe, INSERT sinon)
    Vote vote = voteRepo
        .findByActionnaireAndResolution(actionnaire, resolution)
        .orElse(new Vote());

    // 4. Sauvegarder avec le poids = nombre d'actions de l'actionnaire
    vote.setChoix(request.getChoix());
    vote.setPoids(actionnaire.getNombreActions());
    voteRepo.save(vote);
}

8. Couche Repository — Spring Data JPA

Chaque entité possède son interface Repository qui étend JpaRepository. Spring génère automatiquement les requêtes SQL à partir des noms de méthodes.

Repository Méthode personnalisée clé SQL généré
UserRepository findByUsername(String) SELECT * FROM users WHERE username = ?
ActionnaireRepository findByEntrepriseId(Long) SELECT * FROM actionnaires WHERE entreprise_id = ?
ActionnaireRepository findByUserAndEntreprise(User, Entreprise) SELECT * FROM actionnaires WHERE user_id = ? AND entreprise_id = ?
AssembleeGeneraleRepository findByEntrepriseId(Long) SELECT * FROM assemblees_generales WHERE entreprise_id = ?
AssembleeGeneraleRepository findByActionnaire(Long userId) Requête JPQL — AGs des entreprises où l'user est actionnaire
VoteRepository findByActionnaireAndResolution() Vote d'un actionnaire sur une résolution spécifique
VoteRepository findByActionnaire_User_Username(String) Tous les votes d'un utilisateur par son username

9. DTOs — Data Transfer Objects

Les DTOs sont les objets échangés entre le client et le serveur. Ils ne contiennent jamais les données sensibles (ex: mot de passe haché) ni les collections JPA non sérialisables.

DTO Champs principaux Utilisation
LoginRequest username, password Corps du POST /auth/login
LoginResponse token, username, role, id, nom, prénom Réponse du login
RegisterRequest username, password, nom, prénom, email Corps du POST /auth/register
UserDTO id, username, nom, prénom, email, role Profil utilisateur (sans password)
EntrepriseDTO id, nom, secteur, description, logoUrl CRUD entreprises
ActionnaireDTO id, userId, entrepriseId, nombreActions, userNom, userPrenom, username Actionnaire enrichi
AssembleeGeneraleDTO id, titre, date, lieu, description, entrepriseId, entrepriseNom, resolutionIds[], resolutions[] AG complète
ResolutionDTO id, titre, description, ordre, dateAG, entrepriseId Résolution
VoteDTO id, resolutionId, resolutionTitre, choix, poids Vote exprimé
VoteRequest resolutionId, choix (POUR/CONTRE/NEUTRE) Corps du POST /user/vote
