1. Présentation générale

Le frontend est une application React 18 SPA (Single Page Application) qui permet à deux types d'utilisateurs d'interagir avec le système de gestion des Assemblées Générales :

• Administrateur : crée les entreprises, gère les actionnaires et résolutions, organise les AGs
• Actionnaire (USER) : consulte les AGs auxquelles il est convié et exprime ses votes

🔑
Espace Administrateur
Gestion complète des entreprises, actionnaires, résolutions et assemblées générales. Création d'AGs en 3 étapes guidées.
🗳️
Espace Actionnaire
Tableau de bord personnel, liste des AGs disponibles, vote sur les résolutions (POUR / CONTRE / ABSTENTION), historique des votes.
🔐
Authentification JWT
Connexion sécurisée avec token JWT stocké dans localStorage. Routes protégées selon le rôle (ADMIN / USER).
📱
Interface Responsive
Mise en page avec sidebar fixe, contenu principal adaptatif. Styles CSS en JavaScript (inline styles) pour chaque composant.

4. Système d'authentification (AuthContext)

L'authentification est gérée via un React Context (AuthContext.jsx) qui partage les informations de l'utilisateur connecté à travers toute l'application.

⚙️ Fonctionnement

Saisie credentials
→
Appel POST /api/auth/login
→
Réception JWT + profil
→
Stockage localStorage
→
setUser() → redirect
ÉlémentDescription
userObjet contenant : id, username, role, nom, prénom, token
loadingtrue pendant la vérification initiale du localStorage au chargement
login(username, password)Appelle l'API, stocke le token JWT et le profil, met à jour le state
logout()Vide localStorage, remet user à null → redirection vers /login
useAuth()Hook personnalisé — accès rapide au contexte dans n'importe quel composant
// Utilisation dans n'importe quel composant :
const { user, login, logout } = useAuth();

// Connexion
const data = await login('admin', 'password');

// Vérifier le rôle
if (user?.role === 'ADMIN') { /* accès admin */ }

// Déconnexion
logout(); navigate('/login');

🔒 Persistance de session

Au chargement de l'application, un useEffect lit localStorage.getItem('token') et localStorage.getItem('user'). Si un token existe, l'utilisateur reste connecté sans avoir à se reconnecter (même après actualisation de la page).

🛡️ Composant PrivateRoute

// Protège une page — redirige si non connecté ou mauvais rôle
function PrivateRoute({ children, adminOnly }) {
  const { user, loading } = useAuth();

  if (loading) return null;                         // Attendre la vérif
  if (!user) return <Navigate to="/login" />;      // Non connecté
  if (adminOnly && user.role !== 'ADMIN')             // Pas admin
    return <Navigate to="/user/dashboard" />;

  return children;                                  // Accès autorisé
}

6. Couche API — api.js

Toutes les communications avec le backend sont centralisées dans src/api/api.js. Une instance Axios est créée avec la baseURL et un intercepteur ajoute automatiquement le token JWT à chaque requête. 

// Instance Axios configurée
const api = axios.create({ baseURL: 'http://localhost:8080/api' });

// Intercepteur : ajoute le token JWT à chaque requête
api.interceptors.request.use(config => {
  const token = localStorage.getItem('token');
  if (token) config.headers.Authorization = `Bearer ${token}`;
  return config;
});

// Intercepteur : déconnexion automatique si 401 Unauthorized
api.interceptors.response.use(
  response => response,
  error => {
    if (error.response?.status === 401) {
      localStorage.clear();
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);
